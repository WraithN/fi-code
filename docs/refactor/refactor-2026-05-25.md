## E2E 测试重构：Rust → Python

**处理时间：** 2026-05-25 10:37

**模块：** tests/e2e

**重构动机：**
1. 统一测试语言栈，降低维护成本
2. E2E 测试使用 Python 更利于快速迭代和调试
3. 消除 Rust/Python 测试基础设施的重复

**具体改动：**
1. 保留 `tests/e2e-tui/*.rs` 中的 Rust E2E 测试（作为备份），但将其从 `tests/Cargo.toml` 中移除
2. 创建 `tests/e2e/{common,cli,tui,web}/` 统一目录结构
3. 新增 Python E2E 测试：
   - CLI: 7 个用例（help, version, models, single_command, session, server, web_flag）
   - TUI: 8 个用例（help, version, backend_server, greeting, code_writing, task_splitting, SSE, session）
4. 创建 `tests/e2e/common/` 公共包（constants, fixtures, server, project, mock_ai, pexpect_utils, subprocess_utils 等）
5. 添加 Cargo 桥接：`cargo test --test e2e_all` 自动调用 pytest
6. 修改 `tests/Cargo.toml` 移除 Rust E2E targets，新增 `e2e_all` target 指向桥接脚本
7. 修复 `tests/e2e/conftest.py` 的模块导入路径（`sys.path` 与 `from common.fixtures import *`）

**预期收益：**
- 统一 Python 技术栈，团队只需维护一套 E2E 基础设施
- pytest 生态更丰富（pexpect, playwright, requests）
- 每个用例独立文件，便于单独调试和并行运行
- 双模式触发兼容既有 cargo test 工作流
