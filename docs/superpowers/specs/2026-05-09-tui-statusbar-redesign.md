# TUI 状态栏与布局改版设计文档

> 日期：2026-05-09
> 主题：TUI 底部状态栏全新设计 + 三栏布局优化
> 状态：已评审

---

## 1. 设计目标

重构 fi-code TUI 的底部状态栏，解决以下问题：
1. 品牌标识缺失 — 新增 FiCode 品牌名常驻展示
2. 进度百分比不准确 — 改为纯视觉流式进度条，不做数值估算
3. 布局信息密度低 — 将 Token 统计、耗时、模型信息整合到单行

同时优化整体三栏布局，明确各区域职责。

## 3. 底部状态栏设计（核心改版）

### 3.1 布局格式

```
品牌名 + 纯进度条 ｜ 耗时 ｜ Token进/出 ｜ 当前模型
```

**示例：**
```
FiCode  [████████████░░░░░░] ｜ 耗时：52s ｜ IN:986 OUT:2450 ｜ Model:kimi-code
```

### 3.2 各段说明

| 段 | 内容 | 说明 |
|---|---|---|
| **品牌** | `FiCode` | 状态栏最左端，永久常驻 |
| **进度条** | `[████████░░░░]` | 纯字符可视化，无百分比数字 |
| **耗时** | `耗时：52s` | 当前会话/轮次累计耗时 |
| **Token** | `IN:986 OUT:2450` | 输入/输出 Token 计数 |
| **模型** | `Model:kimi-code` | 当前使用的模型名称 |

### 3.3 进度条逻辑（无百分比）

- **空闲就绪**：整条为空 `[░░░░░░░░░░░░░░░░░░░]`
- **AI 思考/工具执行中**：进度条做**流式动态滚动/逐格填充**，只表达「正在运行」，不代表具体百分比
- **单轮任务结束**：停止动画，保持当前长度，不强制拉满 100%
- **状态视觉反馈**：完全不做数值估算，只做状态指示

### 3.4 状态机

```
空闲 → [░░░░░░░░░░] 静态空条
  │
  ▼ 用户发送消息 / AI 开始生成
运行 → [██░░░░░░░░] 流式动态填充动画
  │
  ▼ 生成完成 / 工具执行完毕
暂停 → [████░░░░░░] 定格在当前长度
  │
  ▼ 新建会话 / 重置
空闲 → [░░░░░░░░░░] 重置为空
```

## 4. 品牌标识设计

- **位置**：底部状态栏**最左端**，永久固定
- **样式**：纯文字品牌名，和状态栏文字字号、配色保持一致，不加粗、不加大、不特殊装饰
- **曝光**：任何布局状态（左侧开/关）都永久可见
- **极简克制**：不单独开 logo 栏、不占用主视觉区，融入功能状态栏

## 5. 其他模块（无改动）

### 5.1 左侧边栏
- Ctrl+B 显隐
- 内部上下结构（上文件树 + 下留白）
- 仅文件树，可伸缩宽度

### 5.2 中间主区域
- 对话 / 思考 / 工具 / Diff
- 长内容自动截断、可折叠展开
- 会话标识：`--[session: #前4位UUID-轮次] --`

### 5.3 右侧边栏
- **常驻不关闭**
- Task 完成情况
- 本次会话变更文件
- 双区块布局不变

### 5.4 Ctrl+L 日志弹窗
- 悬浮唤起
- ESC 关闭
- 可上下滚动

## 6. 全局快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+B` | 左侧文件树 显示/隐藏 |
| `Ctrl+L` | 唤起会话日志悬浮弹窗 |
| `ESC` | 关闭弹窗 / 收起折叠内容 |
| `Enter/Space` | 展开思考、展开截断输出 |

## 7. 实现范围

### 修改文件
- `src/tui/components/status_bar.rs` — 完全重写底部状态栏（品牌、进度条、耗时、Token、模型）

- `src/tui/app.rs` — 更新状态栏调用接口（传入耗时、Token、模型、生成状态）

### 新增数据结构
- `StatusBarState` — 状态栏内部状态（耗时计时器、Token 计数、进度动画状态）
