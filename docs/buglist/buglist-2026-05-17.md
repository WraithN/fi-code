# Bug 记录 — 2026-05-17

## 1. TUI 聊天界面渲染错乱

- **解决时间**: 2026-05-17 16:59
- **模块**: `crates/tui/src/components/chat.rs`
- **现象**: TUI 聊天区域出现内容截断、文本错位、旧帧残留等问题。具体表现为：
  - 消息文本被截断（如 "Python" 显示为 "ython"）
  - 不同回合的内容重叠或夹杂不相关的旧文本片段
  - 界面中出现重复的 `session` 标签和边框残留
- **根因**: `Chat::draw` 方法中多处高度计算的宽度参数与实际渲染宽度不一致，导致 `part_height` 被系统性低估：
  - 用户消息高度：`draw` 中使用 `inner.width - 1`，而实际内容渲染宽度为 `inner.width - 2`（`content_rect` 左右各缩进 2 列）
  - AI Part 高度：`draw` 中使用 `inner.width - 1`，而 `clip_rect_ai` 分配的可用宽度为 `inner.width - 2`
  - `total_height()` 中统一使用 `inner.width - 2` 计算，因此 `total_height` 通常大于 `draw` 实际占用的总高度，导致 `scroll_y` 偏大，视口向下偏移过多，露出旧帧残留
  - 此外，`Block` 边框不会自动清除 `inner` 区域背景，未覆盖区域会保留上一帧内容
- **修复方案**:
  1. 将 `draw` 中所有 `inner.width.saturating_sub(1)` 统一改为 `inner.width.saturating_sub(2)`，与 `total_height` 和实际渲染宽度保持一致
  2. 在 `frame.render_widget(block, area)` 后显式使用 `BgFill` 清除整个 `inner` 区域背景，消除旧帧残留

## 2. TUI 聊天界面出现大量异常空白

- **解决时间**: 2026-05-17 17:15
- **模块**: `crates/tui/src/components/part_renderer/tool_call.rs`
- **现象**: 修复根因 1 后，界面残留问题改善，但聊天内容之间出现大量异常空白行。例如 ToolUse 卡片（如 `handle_task_plan`、`write`）下方有多行空白才出现下一个 Part。
- **根因**: `ToolCallRenderer::height` 使用手动公式计算文本行高：
  ```rust
  h += w / width.max(1) + 1;
  ```
  该公式在文本宽度 `w` 恰好是 `width` 的整数倍时（如 `w == width`）会多算 1 行，导致 `part_height` 被系统性高估。`current_y` 因此跳跃过多，在元素之间留下大片空白。
- **修复方案**: 将手动计算替换为 `Paragraph::line_count(width).max(1) as u16 + 1`，确保 `height` 与 `draw` 中 `Paragraph` 的实际渲染行数完全一致。
- **相关 Commit**: 待提交
